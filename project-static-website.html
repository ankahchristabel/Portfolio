HOSTING A STATIC WEBSITE ON AWS WITH S3 AND CLOUDFRONT:A STEPPING MILESTONE ACHIEVEMENT

•••

2 min read

Back to Projects

On this day, I stepped up to host my very own static website on AWS, leveraging Amazon S3 for storage and CloudFront for global content delivery. It seems simple, but it marked a meaningful milestone in my cloud learning journey—hands-on deployment, HTTPS, caching, and cost-efficiency all working together. Below is a short overview and visual of the result.

Step 1: Create an S3 Bucket

I started by creating an S3 bucket, which is the foundation for my static website. To host the static site, I configured the following settings:

  • Log in: AWS Management Console.
  • Navigate: To S3 and create a new bucket
  • Bucket name: I chose a globally unique name for my bucket.
  • Static website hosting: I enabled static website hosting.

Step 2: Upload Website Files

After the bucket creation, I uploaded all of the HTML, CSS, and JavaScript files for my static website straight into the S3 bucket.

Step 3: Create CloudFront Distribution

To enhance my website’s performance and reliability, I created a CloudFront distribution. In the CloudFront console, I created a new distribution and configured the origin to point to my S3 bucket hosting the static site.

  • Go to CloudFront: Open the CloudFront console and create a new distribution.
  • Origin: Select the S3 bucket as the origin for the distribution.

Step 4: Set Permissions

Allowing read access to the CloudFront Origin Access (OAC/OAI) so objects in the S3 bucket are securely served through CloudFront.

Step 5: Set Up CloudWatch Metrics

To enhance my website’s performance and reliability, I enabled CloudFront metrics in CloudWatch to monitor distribution activity and identify trends or issues quickly.

  • CloudFront metrics: Selected and enabled CloudFront standard metrics in CloudWatch for visibility into requests, errors, and data transfer.

I selected all the available metrics for my CloudFront distribution to get comprehensive visibility.

Step 6: Test the Website

After setting up everything, I tested my website using the CloudFront distribution domain name, and it worked flawlessly. The content was served globally, fast, and securely.

What I Learned Along the Way

The experience of hosting my static website on AWS using S3 and CloudFront was great. It helped me:

  • Understand how AWS S3, CloudFront, and other services work together to deliver fast, reliable, and scalable websites.
  • See how you can scale automatically, reduce latency, and distribute content globally without managing physical servers—a game-changer for developers.
  • Appreciate how a few hurdles can teach more about cloud architecture and the importance of paying attention to detail.

Conclusion

This milestone has been an exciting one in my journey in cloud. I look forward to exploring more AWS services, building more simple systems to complex systems, and expanding my knowledge of cloud-based technologies. Let’s connect and share ideas.
