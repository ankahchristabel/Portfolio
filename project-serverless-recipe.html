HOSTING A SERVERLESS RECIPE-SHARING APPLICATION

•••

4 min read

Back to Projects

On this day, I stepped up to host a serverless recipe-sharing application on AWS, leveraging CloudFront and Amazon S3 as the frontend, API Gateway and AWS Lambda for the backend, Amazon Cognito for authentication and DynamoDB as the data store. This article outlines the deployment process for a serverless recipe-sharing application using AWS CloudFormation templates. We'll delve into the HTTP deployment option and discuss the components it creates to support both frontend and backend functionalities.

Hosting the Static Website

A step-by-step look at how I set up my recipe-sharing application using the http deployment:

Step 1: Deploying the CloudFormation Template

Navigate to the CloudFormation console.

  • Create Stack.
  • Select "Choose an existing template".
  • Select "Upload a template file".
  • Upload the YAML template.

Step 2: Configure Stack details

  • Configure the stack parameters

Step 3: Review and Deploy

  • Review the stack configuration for accuracy.
  • Acknowledge that CloudFormation may create IAM resources.
  • Click "Create stack" to initiate the deployment process.
  • Wait until the stack status changes to "CREATE_COMPLETE".

Step 4: Configuring and Deploying the Frontend

The two configuration files used aws-exports.ts and config.tsx is to configure the aunthentication with data from the Cognito user pool and configure application, namely the API URL respectively.

The two files are in details below:

aws-export.ts

This file is composed of four configuration variables:

  • AWS_PROJECT_REGION: The region you deployed your solution
  • AWS_COGNITO_REGION: Your Cognito user Pool region
  • AWS_USER_POOLS_ID: The ID of your user pool
  • AWS_USER_POOLS_WEB_CLIENT_ID: The client ID of your Cognito User Pool application

configs.tsx

  • CONFIG_API_URL: HttpAPI endpoint

Update the configuration files with the retrieved values from the Outputs Tab and save the changes.

Step 5: Install dependencies and build the Frontend

Once the configurations are set, you'll need to prepare the application files for deployment by initiating the build process to transform the code into a web-ready bundle that can be served to bowsers.

Run the build command for the application in the root of the folder that contains the files. 

npm install && npm run build

A /dist folder is created with the compiled application files. The folder name may but is typically /build or /dist.

Verify the Build Files

Check and ensure the dist folder contain all necessary files for deployment.

Step 6: Uploading to S3

Navigate to the S3 Console

  • Open the S3 Console.
  • Locate the bucket created by the CloudFormation stack [the bucket name follows the naming convention defined in the template].
  • Click Upload.
  • Select the contents of the /dist folder's contents.
  • Click Add Folder and select the assets/ folder.
  • Click Upload to finalize the process

Step 7: Accessing the Application

To access the deployed application, retrieve the CloudFront URL:

  1. Go to the CloudFront console at https://console.aws.amazon.com/cloudfront/.
  2. Click on Distributions on the left menu.
  3. Select the distribution you created.
  4. Copy the Distribution domain name value from the Details section.
  5. Paste the URL into your browser to access the application

Step 8: Testing Authentication and User Management

This application includes authentication via Amazon Cognito.

  • Navigate to the Admin Page.
  • Enter your credentials received via email:
    • Username (defined in the CloudFormation parameters)
    • Temporary password
  • Update your password when prompted.
  • Verify your email address using the verification code received.
  • Once authenticated, explore the Admin Console

Step 9: Exploring Backend API Functionality

  • Navigate to the API Gateway Console
  • Select your deployed API.
  • Click Integrations from the left menu.
  • Explore different API routes such as DELETE /recipes/{recipe_id}.
  • Click on the Lambda function linked to a route to inspect its logic.

Step 10: Securing API Endpoints with Cognito Authorization

  • Navigate to API Gateway Console.
  • Select your API.
  • Click Authorization from the left menu.
  • Verify that routes requiring authentication have the JWT Auth authorizer enabled.
  • Test an authenticated request by including the access token:
curl -i -H "Authorization: Bearer YOUR_ACCESS_TOKEN" YOUR_API_URL/auth
