HOSTING A RECIPE-SHARING APPLICATION:ANOTHER STEPPING MILESTONE

•••

4 min read

Back to Projects

On this day, I stepped up to hosted a recipe-sharing application on AWS, leveraging CloudFront and S3 as the frontend, VPC and EC2 for the backend, and DynamoDB as the data store. It's a relatively simple task but it marked an important milestone in my learning and development as I explored the full potential of cloud services. This article outlines the deployment process for a recipe-sharing application using AWS CloudFormation templates.

Step-by-Step Implementation

Step 1: Configuring the Frontend

Navigate to the CloudFormation console and select my template for http configuration.

  • Create Stack
  • Choose "Upload a template file".
  • Upload the selected YAML template

Step 2: Configure Stack details

  • Provide a Stack name

Step 3: Review and Deploy

  • Review the stack configuration for accuracy.
  • Acknowledge that CloudFormation may create IAM resources.
  • Click "Create stack" to initiate the deployment process.
  • Wait until the stack status changes to "CREATE_COMPLETE".

The CloudFormation template provisions the following resources for the applications frontend, backend, and data storage:

  • Frontend: Amazon S3, Amazon CloudFront
  • Backend: Public EC2 instance
  • Data Store: Amazon DynamoDB, which provides a fully managed NoSQL database to store and retrieve saved recipes efficiently.

Step 4: Preparing the Application Files

The configuration file contains the variables below:

  • CONFIG_MAX_INGREDIENTS
  • CONFIG_MAX_STEPS
  • CONFIG_MAX_RECIPES
  • CONFIG_USER_PAGE_TITLE
  • CONFIG_ADMIN_PAGE_TITLE
  • CONFIG_appConfig
  • API_URL

Update API_URL to match the deployed API endpoint

Step 5: Deploying the Frontend

Build the Application

Once the configurations are set, you'll need to build the frontend. Run the build command for your React application 

npm install && npm run build

A /build or /dist folder is created with the compiled application files.

Verify the Build Files

Check and ensure the dist folder contain all necessary files for deployment.

Step 6: Uploading to S3

Navigate to the S3 Console

  • Open the S3 Console
  • Locate the bucket created by the CloudFormation stack

Upload the Files

  • Select the contents of the /dist folder's contents and upload them to the S3 bucket.

Step 7: Testing and Deployment

Testing the Backend API

Test interactions with the backend services to confirm that the API_URL is correctly configured. To test the API_URL, make requests to specific endpoints. In my case I used cURL to test the connectivity. 

$ curl -i 'https://api.awscloudprojects.site/recipes'

Testing the Frontend

  • Navigate to AWS CloudFront Console.
  • Copy the CloudFront distribution URL.
  • Paste the URL into a web browser to access the application.

What I Learned Along the Way

  • The few hurdles on this journey taught me more about cloud architecture and the importance of paying attention to detail.
  • Whether I used http or https configurations, the resulting infrastructure is tailored to meet my application's specific needs.

Conclusion

This second milestone has been an exciting one in my journey in cloud. I look forward to exploring more AWS services, building more simple systems to complex systems, and expand my knowledge of cloud-based technologies.
